1. PL/SQL 의 특징으로 틀린것은?
    가. 변수, 상수 등을 선언하여 SQL 문장 간 값을 교환한다.
    나. PL/SQL 은 응용 츠로그램의 성능을 향산시킨다.
    다. DBMS 정의 에러나 사용자 정의 에러를 정의하여 사용 할 수 있다.
    라. IF, LOOP 등의 언어를 사용하여 객채지향적인 프로그램이 가능하다. --

2. PL/SQL 의 구조의 설명으로 틀린 것은?
    가. DECLARE : BEGIN ~ END 정레서 사용될 변수와 인수에 대한 정의 및 데이터 타입을 선언하는 선언부이다.
    나. BEGIN ~ END : 개발자가 처리하고자 하는 로직을 선언하여 처리하는 실행부 이다.
    다. EXCEPTION : BEGIN ~ END 절에서 에러가 발생하면 에러를 어떻게 처리할 것인지 정의 하는 예외 처리부이다.
    라. IS : 재사용 하고자 하는 PL/SQL 을 선언 하는 객체 생성부 이다.--

3. PL/SQL 의 기능의 설명으로 틀린것은?
    가. Procedure 에서 사용하는 select 문장은 결과 값이 반드시 있어야 된다.
    나. Procedure 에서 대입 연산자는 ':=' 을 사용한다.
    다. Trigger 는 선언부에서 RollBack Commit 을 통하여 데이터의 무결성을 확보 할 수 있다. --
    라. Function 은 return 을 통하여 하나의 값을 반드시 되돌려줘야 한다.

4. 규칙기반 옵티마이저의 우선 순위 규칙의 설명중 틀린것은?
    가. Single row by rowid : 다른 정보를 참조하여 원하는 행을 엑세스 할 수 있다. --
    나. Single row by unique or primary key : 유일 인덱스를 통해서 하나의 행을 엑세스 하는 방식
    다. Composite index : 복합 인덱스에 동등('=') 조건으로 검색 하는 경우
    라. single column index : 단일 칼럼 인덱스에 동등('=') 조건으로 검색 하는 경우

5. 비용기반 옵티마이저의 설명으로 틀린것은?
    가. 비용기반 옵티마이저는 질의 변환기, 대한 계획 생성기, 비용 예측기 등의 모듈로 구성되어 있다.
    나. 질의 변환기는 인덱스를 사용하는 실행계획을 생성하는 모듈이다.  --
    다. 대안 계획 생성기는 동일한 결과를 생성하는 다양한 대안 계획을 생성하는 모듈이다.
    라. 비용 예측기는 대안 계획 생성기에 으해서 생성된 대안 계획의 비용을 예측하는 모듈이다.

6. 인덱스의 설명으로 틀린것은?
    가. 인덱스는 테이블을 기반으로 선택적으로 생성할 수 있는 구조이다.
    나. 인덱스의 기본적인 목적은 검색 성능의 최적화이다.
    다. 각 테이블은 하나의 인덱스를 생성 할 수 있다. --
    라. insert, update, delete 같은 기능은 오히려 느려질 수 있다.

7. 전체 테이블 스캔이 일어나는 경우로 틀린거은?
    가. SQL 문에 조건이 존재 하지 않는 경우
    나. SQL 문의 주어진 조건에 사용 가능한 인덱스가 존재하지 않는 경우
    다. 조건을 만족하는 데이터가 많을때 옵티마이저의 판단에 의한 경우
    라. 직렬 처리 방식으로 처리 하는 경우  --

8. 인덱스 스캔의 설명으로 틀린것은?
    가. 인덱스를 구성하는 칼럼의 값을 기반으로 데이터를 추출
    나. 유일 스캔은 유일 인덱스를 사용하여 단 하나의 데이터를 추출 하는 방식
    다. 인덱스 구성칼럼중 컬럼이 전부 일치 하지 않고 하나라도 포함되면 그 컬럼값을 기반으로 데이터를 추출 --
    라. 범위 스캔은 인덱스를 이용 하여 한건 이상의 데이터를 추출.

9. 조인 수행 원리중 NL 조인의 순서로 으로 틀린것은?
    가. 1 - 선행 테이블에서 조건을 만족하는 철 번째 행을 찾음
    나. 2 - 선행 테이블의 조인 키를 가지고 후행 테이블에 조인 키가 존재하는 찾으로감.
    다. 3 - 후행 테이블의 인덱스에 선행 테이블의 조인 키가 존재하는 확인
    라. 4 - 인덱스에서 추출한 레코드 식별자를 이용하여 선행 테이블 엑세스 --

10. 조인 수행 원리중 Hash 조인의 순서를 틀린것은?
    가. 1 - 선행 테이블에서 주어진 조건을 만족하는 행을 찾음
    나. 2 - 선행 테이블의 조인키를 기주능로 해쉬 함수를 적용하여 해쉬 테이블을 생성
    다. 3 - 후행 테이블에서 주어진 조건을 만족하는 행을 찾음
    라. 4 - 선행 테이블의 조인 키를 기준으로 후행 테이블의 해쉬 함수를 적용하여 해당 버킷을 찾음 --
